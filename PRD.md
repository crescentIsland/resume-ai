## 一、产品背景

### 问题

准应届生在求职初期面临三种典型困境：

1. **没有简历，不知道从哪里开始写** — 有经历，但不会组织和表达
2. **有简历草稿，但写得不够好** — 不清楚哪里有问题、怎么改才专业
3. **有简历了，但对未来很迷茫** — 不清楚自己离目标岗位差什么、该怎么提升

现有产品的问题：
- 简历工具（如超级简历）：只解决排版，不解决内容质量，也不解决「之后怎么办」
- 招聘平台（如 BOSS 直聘）：信息密度高，以岗位和竞争数据为主，给用户带来焦虑而非行动方向

### 竞品分析结论

通过对 BOSS直聘、超级简历、猎聘、Kickresume、JobRight.ai、Resume.io 的分析，发现：

**所有竞品都在做的事**：简历制作/优化、岗位匹配推荐、求职材料生成

**所有竞品都没做的事**：告诉用户离目标岗位差什么、给出技能提升路径、制定从现在到投递的行动计划

> 所有人都在帮用户找工作，没有人在帮用户准备好自己。

### 机会

用 AI 覆盖「从零写简历 → 优化简历 → 求职规划」的完整路径，做准应届生求职准备阶段的私人顾问。

---

## 二、目标用户

**核心用户**：准应届生（大三/大四/研究生），有明确目标岗位方向，但求职经验不足。

**典型用户画像**：
- 即将秋招/春招，还有 4-8 周准备时间
- 知道自己想做什么方向，但不确定自己够不够格
- 需要的不是焦虑，而是「我接下来该做什么」的确定感

---

## 三、产品定位

> 不管你现在有没有简历，AI 都能帮你走完从「现在的我」到「准备好投递」的每一步。

不是简历生成器，不是招聘平台，是**求职准备阶段的私人顾问**。

## 四、核心功能规划

### v1 三大核心功能

| 功能 | 描述 | 优先级 |
|------|------|------|
| AI 辅助创造简历 | 用户填写经历信息，AI 生成一份专业简历 | P0 |
| AI 辅助修改简历 | 用户上传已有简历，AI 诊断问题 + 改写优化 | P0 |
| 求职规划 | 输入目标 JD + 剩余时间，AI 分析技能 gap + 生成时间表 | P0 |

### v2 规划功能（暂不开发）

| 功能 | 描述 |
|------|------|
| 岗位匹配 | 简历与真实在招岗位的匹配评分（参考 JobRight） |
| 企业招聘入口 | 开放给招聘方，在平台发布岗位 |

## 五、三条用户路径

产品根据用户当前状态分为三条路径，用户进入后先选择自己的情况：

---

### 路径 A：我还没有简历

**用户状态**：有经历，但没有简历，不知道怎么写。

**流程**：
```
填写个人信息
  - 基本信息（姓名、学校、专业、毕业时间）
  - 获得的荣誉/奖项
  - 实习经历（公司、职位、时间、做了什么）
  - 项目经历（项目名、角色、做了什么、结果）
  - 校园经历（社团、志愿、竞赛等）
  - 掌握的技能
  - 目标岗位方向
        ↓
AI 生成一份完整简历
  - 结构清晰、语言专业
  - 重点突出与目标岗位相关的经历
  - 用量化数据强化描述（AI 会引导用户补充数字）
```

**设计原则**：不只是「填空生成」，AI 要像顾问一样，帮用户把平淡的经历变成有说服力的简历语言。

---

### 路径 B：我已经有简历，想优化

**用户状态**：有简历草稿，知道不够好，但不知道怎么改。

**流程**：
```
上传/粘贴现有简历
  ↓
[可选] 粘贴目标岗位 JD
  ↓
AI 输出诊断报告：
  - 简历评分（1-100）
  - 优势亮点
  - 改进建议（具体到每个部分，说明为什么）
  - 改写建议稿（关键段落的专业改写版本）
  - [有 JD 时] 匹配度分析 + 针对性修改建议
```

**设计原则**：建议必须具体。不说「工作经历太模糊」，要说「建议改成'负责 XX，推动 YY 提升了 ZZ%'的结构」。

---

### 路径 C：我已经有简历，想知道下一步怎么做

**用户状态**：简历基本准备好了，但对如何进一步提升自己感到迷茫。

**流程**：
```
上传/粘贴现有简历
  ↓
输入目标岗位 JD
  ↓
输入距投递时间（还有几周）
  ↓
AI 生成求职规划：
  - 技能 gap 分析（对比简历和 JD，列出缺的技能）
  - 推荐学习资源（联网搜索当前高质量课程，注明平台和时长）
  - 分周行动时间表
  - 面试准备建议（常见题型、准备框架）
```

**示例输出**：
```
距投递还有 6 周

技能差距：
  ✓ 产品思维：简历中有体现
  ✗ 数据分析（SQL）：JD 要求，简历未体现
  ✗ 用户研究方法：JD 要求，简历未体现

推荐资源：
  - 《SQL 必知必会》（书，约 15 小时）
  - 用户访谈方法论 — 网易云课堂 XX 课程（6 小时）

行动时间表：
  第 1-2 周：根据 JD 修改简历，补充数据分析相关项目描述
  第 3-4 周：学习 SQL 基础 + 用户研究方法入门
  第 5 周：准备面试，练习 STAR 故事 + 产品方案题
  第 6 周：批量投递，每次面试后复盘迭代
```

## 六、首页设计

**主色调**：#91c53a（活力绿）+ #5e55a2（沉稳紫），不同区域可调整透明度和深浅。

**布局**：
- 顶部固定导航栏：Logo + 三个功能页入口链接
- Hero 区域左右分栏：
  - 左：大号 Slogan + 副标题 + CTA 按钮
  - 右：产品演示截图或操作示意图
- 下方：三大功能卡片简介

## 六、不做什么（范围边界）

- 不做简历排版/模板（不和超级简历竞争）
- 不做招聘信息聚合（不和 BOSS 直聘竞争）
- 不做求职社区/论坛
- v1 不做用户账号系统（无需注册登录）

## 七、成功指标

| 指标 | 说明 |
|------|------|
| 完成率 | 用户选择路径后，有多少人走完了全流程 |
| 有用率 | 结果页「这对你有帮助吗」的正向反馈比例 |
| 分享率 | 用户复制/截图规划的比例（间接反映价值感知） |
| 路径分布 | A/B/C 三条路径各占多少，了解用户真实需求分布 |

## 九、设计决策记录

| 问题 | 决策 | 原因 |
|------|------|------|
| 路径 A 信息填写方式 | 分步骤引导，每步只问一类信息 | 一次性长表单让用户烦躁，放弃率高 |
| 三条路径的入口设计 | 做引导问答（「帮我选」），而非直接展示三个选项卡 | 用户不一定能准确判断自己属于哪条路径，引导问答降低决策成本 |

## 十、开放问题

- 路径 C 的时间表，用户会真的执行吗？是否需要「提醒」机制？（v2 考虑）
- 联网搜索课程的质量如何保证？需要设计过滤策略。
- 引导问答设计：问几个问题合适？问题怎么问不显得啰嗦？
